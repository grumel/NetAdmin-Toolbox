# Security Policy

## Supported versions

| Version | Supported |
| --- | --- |
| 0.6.x | Yes |
| 0.5.x and older | No |

## Reporting a vulnerability

Please do not open a public issue for an undisclosed vulnerability or include secrets, personal data, or exploitable details in a public discussion.

Use GitHub's private security reporting or Security Advisories for this repository when available. If that private channel is not enabled, contact the maintainer through the repository owner's GitHub profile and request a private reporting channel. This repository does not publish a security email address.

Include the affected version, affected module or URL, reproduction steps, impact, and any suggested mitigation. Allow time for investigation and coordinated disclosure before publishing details.

## Scope

Security reports may cover client-side code, service-worker caching, unsafe rendering, dependency or workflow configuration, and accidental disclosure of sensitive data. Please do not report ordinary feature requests or unsupported browser limitations as security vulnerabilities.
